# Airtable-Foam-Submission
This workflow is initiated every time a new submission is received via an Airtable form. It automatically creates a new record, routes the submission to different internal rating paths, consolidates the ratings, and finally uses an AI Agent to perform an analysis or make a final decision, updating the Airtable record with the result.
The project is ideal for teams who need to automate the review and qualification process for form submissions (e.g., job applications, feature requests, content briefs, etc.).

4. üìà Rating Paths (Update Record)
These three parallel paths simulate different team members or systems providing a rating or feedback for the submission:

Video Editor Rating: Updates the record with a rating/feedback from the perspective of a Video Editor.

Graphic Designer Rating: Updates the record with a rating/feedback from the perspective of a Graphic Designer.

AI Manager Rating: Updates the record with a rating/feedback from the perspective of an AI Manager.

5. üß† AI Agent (Core Logic)
Purpose: The central intelligence of the workflow. It takes the original submission data and the ratings from the three paths and uses an AI model to make a decision.

Model: Google Gemini Chat Model (specified as the underlying language model).

Configuration: The Agent uses a defined Chat Model, may leverage Memory to maintain conversation history (if applicable to the use case), and can be integrated with Tools (e.g., search, code execution) to assist with its final analysis.

6. ‚úÖ Update Record (Final Action)
Purpose: Records the final output generated by the AI Agent back into the Airtable record.

Details: This typically updates fields like Status, AI Analysis/Summary, or Final Score.
